## Pedagogical Assessment Framework

Each visual opportunity evaluated against:
1. **Teaching Value**: Does it TEACH a concept or just SHOW data?
2. **Constitutional Alignment**: Does it support AIDD principles and co-learning?
3. **Message Clarity**: Can teaching goal be stated in one sentence?
4. **Non-Redundancy**: Does it add value beyond existing text/tables?

#### Opportunity 5.1: M-Shaped vs T-Shaped vs Specialist Profile
- **Location**: Section "Beyond T-Shaped: Understanding M-Shaped"
- **Format**: Three profile silhouettes showing skill depth patterns
- **Pedagogical Assessment**: **PASS** ✓
- **Teaching Goal**: "Students visually distinguish M-shaped (multiple deep peaks) from T-shaped (one deep peak) and specialist (one narrow peak) profiles"
- **Constitutional Alignment**: Supports Three-Role AI Partnership (Principle 18) by showing how AI enables multi-domain depth
- **Design Approach**:
  - Three side-by-side profile graphs
  - X-axis: Different domain areas (labeled generically: "Domain 1", "Domain 2", etc.)
  - Y-axis: Skill depth (Beginner → Expert)
  - Specialist: One tall narrow bar
  - T-Shaped: One tall bar + several short bars
  - M-Shaped: Multiple tall bars (showing depth in 2-4 domains)
  - Simple bar chart style (not actual human silhouettes)
  - Polar Night palette (navy background, teal/orange bars)
- **Chapter 1 Learnings Applied**:
  - Maximum 3 profiles shown (not 4+)
  - Clear labels on axes (hyphenated compound words)
  - No decorative elements (focus on concept)
- **Filename**: `m-shaped-developer-profile.png`
- **Alt Text**: "Comparison of developer skill profiles: Specialist shows deep expertise in one domain, T-Shaped shows deep expertise in one domain with basic knowledge across others, M-Shaped shows deep expertise across multiple complementary domains"
